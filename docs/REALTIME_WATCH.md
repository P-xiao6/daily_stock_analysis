# 重点盯盘 / 实时建议

`/realtime-watch` 用于盯 1 到 5 只重点股票。它适合盘中少量标的观察，不替代普通自选股看板，也不接券商实盘交易或自动下单。

## 实时动态建议和 AI建议的区别

- 实时动态建议：由行情规则快速生成，例如突破昨日高点、跌破 5 日线、量比超过阈值、接近止损位。刷新行情时只计算规则，不调用 LLM。
- AI建议 / DecisionSignal：来自深度分析报告或 Agent 分析结果，会消耗模型 token，并写入历史报告和决策信号。

如果实时动态建议和上次 AI建议冲突，页面会提示“实时状态与上次 AI建议不一致，建议重新 AI复核。”这表示旧建议可能已失效，但不会自动下单。

## 设置重点股票

打开左侧菜单“重点盯盘”，输入股票代码后添加。页面最多允许 5 只；更多股票请使用普通“自选股”页面，避免行情源压力和误触发高成本分析。

未登录用户只能查看，不能添加、删除、修改规则或触发 AI复核。生产环境建议启用 `ADMIN_AUTH_ENABLED=true`。

## 关键位和规则

每只股票可配置：

- 关键压力位
- 关键支撑位
- 止损位
- 目标位
- 量比阈值
- 涨跌幅提醒阈值
- 是否开启重大规则触发后的自动 AI复核
- AI复核冷却时间
- 每日 AI复核上限
- 默认分析策略

这些配置保存在后端 `realtime_watch_profiles` 表，不使用前端 `localStorage`。

## 刷新规则

支持 30 秒、60 秒、3 分钟、5 分钟刷新，默认 60 秒。页面不可见时前端会暂停刷新；恢复可见时重新拉取快照。

自动刷新只更新：

- 实时行情
- 均线和历史高低点派生字段
- 规则型动态建议
- 最新 active AI建议展示

自动刷新不会调用大模型。

## AI复核

每行都有“AI复核”按钮。点击后可选择：

- 分析策略
- 模型策略：Flash / Pro / 自动
- 是否保存为该股票默认策略

AI复核会进入现有分析任务队列，完成后生成历史报告，并由既有分析链路写入 DecisionSignal。页面会轮询任务状态并刷新最新 AI建议。

默认限制：

- 同一股票 AI复核冷却 30 分钟。
- 每只股票每天最多 AI复核 3 次。
- 未登录用户不能触发。
- 行情自动刷新不触发完整 AI 分析。

## 为什么不建议每分钟 AI 分析

盘中行情变化很快，但大模型分析成本高、耗时长，而且低配服务器并发能力有限。每分钟自动 AI 分析会带来：

- token 成本快速上升
- 分析队列堆积
- 服务器 CPU/内存压力
- 数据源频繁请求
- 多份报告相互覆盖，反而降低决策清晰度

更稳妥的方式是：用规则做高频筛选，只在跌破关键位、放量异动、AI建议可能失效等场景手动 AI复核。

## 2核2G 推荐配置

- `MAX_WORKERS=1`
- 默认刷新间隔 60 秒
- 重点盯盘不超过 5 只
- 自动 AI复核默认关闭
- 每只股票每天最多 AI复核 1 到 3 次
- 不运行本地大模型
- 使用云端 API

## API 摘要

- `GET /api/v1/realtime-watch/profiles`：读取重点盯盘配置。
- `POST /api/v1/realtime-watch/profiles`：新增重点股票。
- `PUT /api/v1/realtime-watch/profiles/{stock_code}`：更新规则。
- `DELETE /api/v1/realtime-watch/profiles/{stock_code}`：删除重点股票。
- `GET /api/v1/realtime-watch/snapshot`：刷新行情和动态建议，不调用 LLM。
- `POST /api/v1/realtime-watch/profiles/{stock_code}/ai-review`：手动提交 AI复核任务。
